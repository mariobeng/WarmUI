🎯 第一层：基础感官补偿

解决"看不见、点不准、听不清"的问题 — 底层能力，也是每个组件库的起点。

<warm-button> 防抖触控按钮

L1 感官补偿
老人手抖导致误触时，防抖延迟 + 触控容差区域能有效降低误按率。开启「适老化模式」体验效果。
标准按钮 适老化按钮
<!-- 适老化按钮：防抖300ms + 容差12px -->
<warm-button elder-friendly debounce-delay="300" tolerance="12">确认挂号</warm-button>

<warm-text> 动态字号引擎

L1 感官补偿
根据内容语义智能分级，保持信息层级的同时放大可读性。支持 heading / body / button / caption 语义级别。
标题级别文字（heading）
正文级别文字（body）- 默认阅读字号
按钮级别文字（button）
辅助说明文字（caption）- 最小字号
当前语义级别：
动态预览文本 — 修改上面的控件试试效果
<warm-text size="heading">标题文字</warm-text>

L1 感官补偿
不仅仅是 TTS，还能自动分析页面结构，用自然语言告诉老人"这个页面有3个部分……"。下方体验文本朗读和页面大纲朗读。
朗读介绍
朗读页面结构

🧠 第二层：认知辅助

老人不是"不想用"，而是"看不懂" — 这是 WarmUI 超越现有组件库的核心层。

<warm-translator> 大白话翻译器

L2 认知辅助
悬停或长按复杂文案，自动弹出"说人话"气泡。把官方用语转为老人能听懂的语言。
请您核实身份信息以确保账户安全。
请确认提交信息的准确性后执行后续操作。
💡 鼠标悬停或长按文字，查看"说人话"效果
<warm-translator translation="就是让您确认是本人操作">
  请您核实身份信息以确保账户安全。
</warm-translator>

💬 第三层：情感连接

技术有温度 — 这是 WarmUI 名称的由来。让每一个操作都传递尊重与关怀。

<warm-note> 家人的便签

L3 情感连接
子女在应用的任意页面给父母留下便签。点击按钮创建一个示例便签。
明天上午 10 点记得去医院复查哦！ 女儿 小美
添加鼓励便签 ❤️ 添加提示便签 💡

📋 一键合规检查

L4 开发者工具
自动检查页面是否满足无障碍规范：触控区域 ≥44x44dp、对比度 AAA 级、图片 alt 替代文本、年龄歧视逻辑。
运行合规检查 ▶
import { runAccessibilityCheck, formatReport } from '@warm-ui/components';
const report = await runAccessibilityCheck();
console.log(formatReport(report));
